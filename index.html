Free Ways to Deploy
Your Tools & HTML

From quick demos to live sites — a curated guide to hosting your Claude-built tools without spending a dime.

Static HTML / CSS / JS

GitHub Pages Best for static files

The go-to storefront for code already in GitHub's warehouse. Push your files to a repo, flip one setting, and it's live at username.github.io/repo-name. Zero cost, custom domain support, handles traffic well.

→ pages.github.com Docs
Netlify Drag & drop deploy

The modern upgrade — drag-and-drop your HTML folder onto their site and it's live in seconds. Friendlier than GitHub Pages for pure static files, and gives you a shareable URL instantly.

→ netlify.com Docs
Vercel Great for React too

Similar to Netlify with a generous free tier. Shines more when you eventually add a framework like React, but handles plain HTML just fine. Deploy from GitHub in one click.

→ vercel.com Docs
CodePen / JSFiddle / StackBlitz Quick demos & sharing

These are whiteboards, not storefronts — great for demos, sharing a single tool, or fast iteration. CodePen especially shines for polished one-page tools and is very shareable.

→ CodePen JSFiddle StackBlitz

How to Use GitHub (Step by Step)

1
Create a free account

Go to github.com and sign up. Your username becomes part of your public URL, so pick something recognizable.

2
Create a new repository

Click the + icon → New repository. Give it a name, set it to Public, and check Add a README file. Think of a repo as a project folder in the cloud.

3
Upload your HTML file

Inside the repo, click Add file → Upload files. Drag your .html file in. Name it index.html if you want it to be the default page. Commit the changes.

4
Enable GitHub Pages

Go to Settings → Pages. Under Source, select your main branch and click Save. GitHub will give you a live URL at username.github.io/repo-name within a minute or two.

5
Update your file anytime

Navigate to the file in your repo, click the pencil ✏️ icon to edit, or upload a new version. Every save ("commit") is version-controlled — you can always roll back. Think of commits as save points in a video game.

6
(Optional) Install Git locally

For faster workflows, install Git and use the terminal: git clone your repo, edit files locally, then git push to publish. Steeper learning curve but much faster once comfortable.
